# delivery-prep

Prepare the finished product for buyer delivery. Sets correct sharing/access permissions, generates download files where required, and organises everything in `products/{slug}/delivery/`. Called by build-agent before the final visual pass.

---

## Input

- `product_type` — string identifying the product type
- `delivery_format` — from poc-brief (e.g. `google-sheets-link + xlsx-download`, `notion-template-link`, `canva-template-link + pdf-export`)
- `working_link` — link to the finished product
- `slug` — product slug (used for file naming and output path)
- `product_name` — product title (used for file naming)

---

## Steps by product type

_To add a new product type: duplicate the nearest section below and adapt it._

### Google Sheets

**Delivery format:** `google-sheets-link + xlsx-download`

1. **Set sharing permissions:**
   - This cannot be done via MCP. Ask the human: "Please open the Google Sheet at `{working_link}`, go to Share → Change to 'Anyone with the link' → set to Viewer, then paste the shareable link back."
   - Wait for the human to share the view-only link. Set `primary_link` to the received link.

2. **Confirm XLSX exists:**
   - The XLSX was already generated by the build script at `products/{slug}/delivery/{slug}.xlsx`. Verify the file exists locally before continuing.

3. **Verify:**
   - Ask the human to open the sharing link in an incognito window and confirm it loads as view-only with all tabs visible and no edit access.

4. **Write delivery manifest** (`products/{slug}/delivery/delivery.json`):
   ```json
   {
     "product_type": "google-sheets",
     "delivery_format": "google-sheets-link + xlsx-download",
     "primary_link": "https://docs.google.com/spreadsheets/d/...",
     "files": [
       { "filename": "{slug}.xlsx", "path": "products/{slug}/delivery/{slug}.xlsx", "format": "xlsx" }
     ],
     "sharing": "anyone-with-link-view",
     "buyer_instructions": "Click the link to open in Google Sheets. Go to File → Make a copy to save your own editable version."
   }
   ```

### Notion

**Delivery format:** `notion-template-link`

1. **Set page as a template:**
   - This cannot be done via MCP. Ask the human: "Please open the Notion page at `{working_link}`, go to Share → Publish as template (or ... → Connections → Template), enable it, and paste the public template link back."
   - Wait for the human to share the template link. Set `primary_link` to the received link.

2. **Verify:**
   - Ask the human to open the template link in an incognito window (logged out of Notion) and confirm the "Duplicate" button is visible and functional.

3. **Write delivery manifest** (`products/{slug}/delivery/delivery.json`):
   ```json
   {
     "product_type": "notion",
     "delivery_format": "notion-template-link",
     "primary_link": "https://www.notion.so/templates/...",
     "files": [],
     "sharing": "public-template",
     "buyer_instructions": "Click the link. Click 'Duplicate' to add this template to your Notion workspace. A free Notion account is required."
   }
   ```

### Canva

**Delivery format:** `canva-template-link + pdf-export`

1. **Set as a shareable template:**
   - This cannot be done via MCP. Ask the human: "Please open the Canva design at `{working_link}`, click Share → Share a link → set to 'Can use as template', and paste the template link back."
   - Wait for the human to share the template link. Set `primary_link` to the received link.

2. **Export PDF:**
   - Use `mcp__canva__export-design` to export all pages as PDF. Save to `products/{slug}/delivery/{slug}.pdf`.

3. **Verify:**
   - Ask the human to open the template link in an incognito window and confirm "Use template" is visible and the design loads correctly.
   - Confirm the PDF file exists at `products/{slug}/delivery/{slug}.pdf` and all pages are present.

4. **Check file size:**
   - Confirm the PDF is under 20MB (Etsy file size limit).
   - If over 20MB: re-export at lower quality or split into parts. Note in delivery manifest.

5. **Write delivery manifest** (`products/{slug}/delivery/delivery.json`):
   ```json
   {
     "product_type": "canva",
     "delivery_format": "canva-template-link + pdf-export",
     "primary_link": "https://www.canva.com/design/.../view?utm_content=...&utm_campaign=designshare&utm_medium=link&utm_source=editor",
     "files": [
       { "filename": "{slug}.pdf", "path": "products/{slug}/delivery/{slug}.pdf", "format": "pdf", "size_mb": 4.2 }
     ],
     "sharing": "canva-template",
     "buyer_instructions": "Click the Canva link. Click 'Use template' to open your own editable copy. A free Canva account is required. A PDF preview is also included."
   }
   ```

---

## Output

```
products/{slug}/delivery/
  delivery.json          ← delivery manifest (all types)
  {slug}.xlsx            ← Google Sheets only
  {slug}.pdf             ← Canva only
```

---

## Notes

- The `primary_link` in delivery.json is the link buyers receive when they purchase on Etsy. Confirm it is correct and publicly accessible before saving.
- For Google Sheets: buyers should always get a view-only link and make their own copy — never share an editable original.
- For Notion: if the workspace doesn't support public templates on the current plan, note it in delivery.json and flag to the human before proceeding.
- For Canva: PDF is a bonus download (for preview/print use). The Canva template link is the primary product.
- After writing delivery.json, confirm the primary_link opens correctly in a browser before closing.
